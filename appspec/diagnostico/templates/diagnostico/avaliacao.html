📖 Como Ler e Interpretar Estas Curvas Aula 5 — Prof. Ronaldo
📉 1. Curva ROC (Receiver Operating Characteristic)

A curva ROC mostra a relação entre Sensibilidade e Taxa de Falso Positivo para todos os limiares de decisão possíveis (de 0.0 a 1.0).

  • Eixo Y = Sensibilidade (TPR) — "De todos os doentes, quantos o modelo detectou?"
  • Eixo X = Taxa de Falso Positivo (FPR) — "De todos os saudáveis, quantos foram alarmes falsos?"
  • Linha diagonal pontilhada = classificador aleatório (jogar moeda). Todo modelo útil deve estar acima desta linha.
  • Canto superior esquerdo = classificador perfeito (Sensibilidade=100%, FPR=0%). Quanto mais a curva se aproxima deste canto, melhor.
  • Ponto vermelho = limiar ótimo encontrado pelo índice de Youden (J = Sensibilidade − FPR).
📐 2. AUC-ROC (Área Sob a Curva)

O AUC resume o desempenho global do modelo em um único número de 0 a 1:

AUC-ROCClassificaçãoInterpretação
0.90 — 1.00ExcelenteDiscriminação quase perfeita
0.80 — 0.89BomBoa capacidade discriminatória
0.70 — 0.79RegularCapacidade moderada
0.50 — 0.69FracoPouco melhor que o acaso
0.50InútilEquivale a jogar moeda

Neste estudo: SVM (AUC=0.92) tem discriminação excelente, KNN (AUC=0.84) é bom, e Alvarado (AUC=0.72) é apenas regular — mostrando que os modelos de ML superam o escore clínico tradicional.

📊 3. Curva Precision-Recall (PR)

A curva PR é mais informativa que a ROC quando as classes são desbalanceadas (neste dataset: 59% apendicite vs 41% sem).

  • Eixo Y = Precision (VPP) — "Quando o modelo diz 'apendicite', com que frequência acerta?"
  • Eixo X = Recall (Sensibilidade) — "De todos os casos reais de apendicite, quantos foram encontrados?"
  • Linha horizontal pontilhada = baseline (prevalência da doença). Todo modelo deve estar acima desta linha.
  • Average Precision (AP) = área sob a curva PR. Análogo ao AUC-ROC mas mais exigente com classes desbalanceadas.
  • Canto superior direito = perfeição (encontra todos os casos E nunca erra). Quanto mais a curva se estica para este canto, melhor.
🎯 4. O que é o Limiar de Decisão e por que ele importa?

Passo 1 — Entenda o limiar: Quando o modelo diz "probabilidade de apendicite = 72%", como ele decide se é apendicite ou não? Ele compara com um limiar (ponto de corte). Se a probabilidade for ≥ limiar, o modelo diz "apendicite". Se for < limiar, diz "não apendicite".

Passo 2 — O padrão do scikit-learn: Por padrão, o scikit-learn usa limiar = 0.50 (50%). Ou seja: qualquer paciente com probabilidade ≥ 50% é classificado como apendicite.

Passo 3 — O que a Curva ROC revela: A curva ROC testa todos os limiares possíveis (de 0.0 a 1.0) e mostra qual dá o melhor resultado. O ponto vermelho no gráfico é o limiar de Youden — o ponto que maximiza o equilíbrio matemático entre Sensibilidade e Especificidade.

Passo 5 — Por que o APPSPEC NÃO usa o limiar de Youden?

🏥 Contexto clínico decisivo: Em triagem de apendicite pediátrica, os dois tipos de erro têm custos muito diferentes:
Falso Negativo (FN) Criança com apendicite vai para casa → risco de perfuração e peritonite GRAVE
Falso Positivo (FP) Criança saudável faz exames adicionais → desconforto, mas sem risco de vida Menor
Como o custo de um FN é incomparavelmente maior, o APPSPEC mantém o limiar 0.50 para priorizar a sensibilidade (não perder doentes).
📖 Lição pedagógica (para a prova):
  1. O limiar de Youden é o ponto matematicamente ótimo — o ponto vermelho na curva ROC acima.
  2. Ele não deve ser aplicado automaticamente. O cientista de dados precisa avaliar o custo dos erros no contexto do problema.
  3. Em triagem médica (detectar doentes): preferir limiar mais baixo → mais sensibilidade.
  4. Em confirmação diagnóstica (evitar cirurgias desnecessárias): preferir limiar mais alto → mais especificidade.
  5. A Curva ROC mostra todas as opções; o especialista do domínio é quem decide qual usar.
Ref: Youden W.J. (1950). Cancer, 3(1), 32-35. | Fawcett T. (2006). DOI:10.1016/j.patrec.2005.10.010
🏆 5. Como Comparar os Modelos nos Gráficos — Passo a Passo

Olhe o gráfico ROC Comparativa acima e siga estes passos:

👉 Passo A — Onde está a perfeição? O canto superior esquerdo do gráfico ROC é o ponto perfeito (detecta 100% dos doentes com 0% de alarmes falsos). O modelo cuja curva chega mais perto deste canto é o melhor.

👉 Passo B — Compare as curvas visualmente: No gráfico, veja que a curva roxa (SVM) está mais acima e à esquerda que a azul (KNN), que por sua vez está acima da verde (Alvarado). Isso significa: SVM > KNN > Alvarado.

👉 Passo C — Confirme pelo AUC na legenda: O número entre parênteses na legenda confirma a ordem: SVM (AUC=0.923) > KNN (AUC=0.838) > Alvarado (AUC=0.719). Quanto maior o AUC, melhor.

👉 Passo D — Repita para a curva PR: Na curva Precision-Recall, a lógica é a mesma: a curva mais acima e à direita vence. Veja que a ordem se mantém: SVM (AP=0.957) > KNN (AP=0.865) > Alvarado (AP=0.759).
